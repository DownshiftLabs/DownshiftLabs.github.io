Some quick facts about the project:

Rails Canonical

Canonicalization of URLs is a huge topic for SEO professionals but very few frontend developers seem to take the precautionary steps in order to avoid large losses for the client. Having seen one too many Rails applications suffering from this, I decided to build a quick, conservative-by-default, configurable implementation of the canonical tag developers can drop in now and worry about later.

Though not as large of a threat as it is now, Google and other SEs used to punish websites found distributing same or similar content under multiple URLs. This rails gem helps by:

  • Enforcing trailing slashes for index actions
  • Serving canonical URLs while ignoring parameters by default
  • Whitelist-configurable to specify extra collection routes
  • Designed to shield against duplicate content attacks
